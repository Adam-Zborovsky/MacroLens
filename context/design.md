# Design System Specification: The Forensic Lens

> This design system was auto-generated by Stitch MCP for the MacroLens project. It establishes the "Optical Precision" and "Forensic Lab" visual identity.

## 1. Overview & Creative North Star: "The Clinical Observer"
The Creative North Star for this design system is **The Clinical Observer**. We are moving away from "consumer tech" friendliness and toward the cold, hyper-precise aesthetic of a high-end forensic laboratory. The interface should feel like a high-performance optical instrument—an extension of the eye that provides data clarity through layered depth and technical transparency.

To break the "template" look, we utilize **Intentional Asymmetry** and **Optical Layering**. We avoid centered, balanced layouts in favor of data-heavy sidebars, offset reticles, and overlapping glass panels that suggest a sophisticated, multi-lens view of the information.

## 2. Colors: Tonal Precision
The palette is rooted in the deep void of a dark lab, utilizing `background` (#0c1324) as the primary canvas.

### The "No-Line" Rule
Traditional 1px solid borders for sectioning are strictly prohibited. Boundaries must be defined through **Tonal Transitions**. Use `surface-container-low` for large secondary regions and `surface-container-high` for interactive zones. Separation is achieved through the contrast between these deep navy tones, never through a "stroke."

### Surface Hierarchy & Nesting
Treat the UI as a series of physical optical slides.
*   **Base Layer:** `surface` (#0c1324) – The laboratory floor.
*   **The Workstation:** `surface-container-low` (#151b2d) – Large layout blocks.
*   **The Specimen:** `surface-container-highest` (#2e3447) – Focused data modules.
*   **Active Overlays:** `surface-bright` (#33394c) – Elements demanding immediate optical focus.

### The Glass & Gradient Rule
To achieve "Optical Precision," use **Glassmorphism** for all floating panels. Apply a `20px backdrop-blur` combined with a semi-transparent `surface-variant` fill. 
*   **Signature Texture:** Use a subtle linear gradient on CTA elements, transitioning from `primary` (#4be277) to `primary-container` (#22c55e) at a 135-degree angle. This mimics the light refraction found in high-end camera lenses.

## 3. Typography: Technical Authority
We pair the utilitarian clarity of **Fira Sans** with the monospaced precision of **Fira Code**.

*   **Display & Headlines (Fira Sans/SpaceGrotesk):** Use `display-lg` (3.5rem) for high-impact data points. Keep tracking tight to maintain a clinical, dense feel.
*   **The Technical Readout (Fira Code):** All numerical data, timestamps, and "scanning" metadata must use Fira Code (mapped to `label-md` or `body-sm`). This reinforces the forensic identity.
*   **Body Text (Fira Sans/Inter):** Use `body-md` for descriptions. The high x-height of the sans-serif ensures readability against the complex backdrop blurs of the "glass" panels.

## 4. Elevation & Depth: The Stacking Principle
Depth is not simulated with shadows; it is earned through **Tonal Layering**.

*   **Ambient Shadows:** If a panel must float (e.g., a critical diagnostic pop-up), use a `primary-fixed-dim` tinted shadow at 4% opacity with a `48px` blur. This creates a "glow" rather than a shadow, suggesting the interface is projected or backlit.
*   **The Ghost Border:** For accessibility in high-density data areas, use a "Ghost Border." This is a 1px border using `outline-variant` (#3d4a3d) at **20% opacity**. It should feel like a hairline trigger on a microscope—visible only when looked for.
*   **Technical Reflections:** Apply a top-down `1px` inner-stroke of `primary-fixed` (#6bff8f) at 10% opacity to "glass" containers to simulate the edge-lighting of a glass slide.

## 5. Components: Forensic Primitives

### Buttons & CTAs
*   **Primary (The "Execute" Action):** Solid `primary-container` (#22c55e) with `on-primary-container` text. No rounded corners (`0px`). 
*   **Secondary (The "Scan" Action):** `outline` ghost button with a `20px` backdrop blur.
*   **States:** On hover, primary buttons should trigger a "flicker" animation (0.1s opacity shift) to mimic technical hardware engagement.

### Input Fields & Technical Reticles
*   **Input Fields:** Forgo the box. Use a bottom-border only (`outline-variant` @ 40%) with Fira Code for input text.
*   **Reticles:** Use custom SVG decorative corners on all major image/video containers to simulate a viewfinder. These should use the `primary` color at low opacity.

### Cards & Lists
*   **The "No-Divider" Rule:** Vertical white space (using the `8` to `12` spacing tokens) is the only acceptable way to separate list items. If separation is visually required, use a subtle background shift to `surface-container-lowest`.
*   **Scanning Animation:** Active list items or cards should feature a horizontal "scan-line" (a 2px tall `primary` gradient) that translates Y-axis 0% to 100% on a loop.

### Diagnostic Tooltips
*   **Style:** Minimalist, using `surface-container-highest` and Fira Code `label-sm`. Sharp 0px corners are mandatory.

## 6. Do’s and Don’ts

### Do:
*   **Use Intentional Asymmetry:** Align technical data to the right and primary visuals to the left to create a "heads-up display" (HUD) feel.
*   **Embrace Monospacing:** Use Fira Code for any string involving numbers or coordinates.
*   **Layer Glass:** Allow background elements (like blurred reticles or map data) to bleed through foreground panels using `backdrop-blur`.

### Don't:
*   **Don't Round Corners:** All `borderRadius` values must remain at `0px`. Roundness conveys "consumer softness," which contradicts "Forensic Lab" precision.
*   **Don't Use Drop Shadows:** Avoid black `rgba(0,0,0,0.5)` shadows. Use tonal shifts or light-tinted ambient glows.
*   **Don't Over-Saturate:** Keep the UI predominantly deep navy (`background`). Reserve the `emerald green` (#22C55E) strictly for successful states, CTAs, and active technical readouts.
